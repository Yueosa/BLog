## 🖥 技术选型

既然是要写一个自己的**博客系统**, 还要有评论区, 撰写文章, 后台管理... ...

所以当然要选择我最熟悉的 `前端 - 后端 - 数据库` 架构啦!

### | 后端

> 我不想用 `python-fastapi` 啦! 这次能不能写点更有挑战性的东西? 比如用 `Rust` 来做后端?

##### Rust + Axum

我想使用 `Rust` 语言和 `Axum` 来作为 **Web框架**

然后再使用 **JWT (JSON Web Token)** 来做无状态登录

### | 前端

> 传统的 `HTML` + `CSS` + `JS` 是我的舒适区, 所以我当然要给自己添麻烦, 所以这次挑战一些新东西吧!

##### Astro + Vue3 + Tailwind

传统 `Vue` 对 **SEO (搜索引擎抓取)** 很不友好, `Astro` 在构建时会尽量生成静态 `HTML`, 剥离 `JS`, 这是目前的最佳实践!

### | 数据库

> 这次依旧使用 `PostgreSQL` 可爱大象数据库~
>
> 增加熟练度!

##### 核心设计思路

1. **Slug (URL别名):** 文章不用 ID 做 URL, 而是使用英文别名, 对 SEO 友好
2. **Soft Delete (软删除):** 加上 `delete_at` 字段, 误删可以恢复
3. **Audit Fields:** 所有表都有 `created_at` 和 `updated_at`
   - 跟学长学的喵
